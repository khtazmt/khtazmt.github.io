	- 笔记流程
	  heading:: true
		- 以记录有意思或可能有意义的变化为主
		  collapsed:: true
			- 先有常规（包括不必记的“没做的事”和“仍然经常发生但可以通过培养习惯改善的行为”），然后记录变化（“新的”、“第一次”、“更快更高更强”等），这样节省时间精力
			  collapsed:: true
				- ^^任何一个发现，即使是天才的发现，发现之前的各种推理假想如果都记下来，那末，废料和各种各样愚蠢荒谬的推断，数量之多会使我们大吃一惊。——《奇特的一生》^^
				  collapsed:: true
					- 但也不要轻视标准化的“流水账”
					  collapsed:: true
						- 记各种事件、东西都很难写全，所以充当自己的记忆的线索或钥匙的简单“流水账”并非不可取
			- 可以补记

		- 记分折移删
			- 记录
				- 碎片笔记
					- 一般而言，为写作记下来的碎片想法就是碎片笔记/草稿
					- 一部分是这样的：自深海上浮、随水压降低而膨胀的气泡在水面裂开，一般最多能看到表层水上部现象的你有了想法，把它记了下来
					- 简短但繁多的碎片笔记可在同一条内的括号（起分隔和分级的作用）里写简单的分类名，没想起怎么分类先记着，可通过引用补充，有空再整理
					- 感言
						- 平均大概十到二十个字吧，费时的是找到最合适的地方（之前没用明白折叠，有不少近义词的分类）塞进去和改写融入
						- 如果用文件夹（页面间）纯文本（页面内）笔记，你可能把一页标题作为一个主题或分类（——名词），要在一种隐含的框架（哦，文件夹改名也是件难事，可能有些人成百上千篇笔记没一个文件夹）中改起名来不太方便（毕竟没有双链，落单了谁记得你是谁？/狗头），而大纲试图能在页面内更快捷地容纳更多分类层级，从而能在单个页面内容纳更多内容——但不会用折叠的话，这些展开的分类一旦超出一张全屏截图的视线范围，没有照相机记忆力的人就会忘记原分类，然后往周围乃至其他页面堆近义词（甚至同一个词在同一层级的不同地方出现两次以上），之后再有像深海浮上来的气泡的同一分类碎片笔记就可能堆积到不同近义词分类的不同层级下并展开占用屏幕空间——（就像俄罗斯方块）——久之整理效率就会显著下降——“所以说，分类很重要”
						  collapsed:: true
							- 不沉淀整理就相当于俄罗斯方块乱搭，堆得越多，空得越多，很快就完了
							- 搭得好，沉淀在消失，乃至几层几层地消，分数却上去了
							- 不是为了输而玩的，而是享受游戏过程，结束一至几局后如果不极度热爱这游戏就能够走出来切换到下一项有趣的活动
				- 微信文件传输助手
					- 一不留神就可能发到其他窗口去了（我目前有两次问题不大的，第一次是发了不少有关正体验着的事件的即时心得给一位老网友，第二次是看神州十二号对接时看弹幕生成一个“实况是否等于直播”的问题草稿发给了爸爸），现在只用来转发一些可能继续转发的块状链接，笔记记到微信笔记里
			- 分类
				- 要“拆分”——复制来复制去的那种“整理”没意思
				- 评论网友
					- 你这标题过于零碎，整体结构过于扁平，能得到什么呀/再见
				- 一般情况下大多数被拆的工作学习等实用类笔记没有像年鉴和日记一样尽量不变的纪念价值
				- 前期通过复制粘贴等方式堆积草稿，一般因为还没有足够丰富的材料和动机使整理工作相对高效地启动——“分类得以高效运行往往恰恰靠前期堆积的有差异的概念，堆积得越多，差异就越明显越细分，分类也就更快更高效——世间无双全法”
					- “完成20％，把剩下80％攒起来，给未来兴趣漂移的新老同伙处理”
			- 折叠
			  collapsed:: true
				- 稍大的主题必然面临超出屏幕的尚不成熟的具体分类，不折叠，这些近义词句就会越堆越多显得无效率，通过折叠将其在视线中聚拢的操作建议别太晚——再分类往往以近义词/同义词比较为主，可能相当费时费神
			- 移动
			  collapsed:: true
				- 移动到需要的地方（如果数量少的话可以直接找到目的地记下）
				  collapsed:: true
					- 拆解
					  collapsed:: true
						- “任务或烦恼”不要让它保持模糊的一团，把它们一条条写下来。你会发现轻重缓急，从而整体上可能舒心不少
			- 删除
			  collapsed:: true
				- 完成的及时删（移走的基本直接删了；日记等的原版可能留着），像todo那样点完成删除线还是在上面，算不得真正删，所以删还是不删还需要再看一遍这不多此一举么？（当然，如果是需要报送完成情况给别人的那就还行）
		- 实践
			- 笔记是为了当类似日记和相册，还是说只把它，我觉得只会这么觉得很难，至少没有物尽其用、浪费时间？至少它们
			- 不要“等”计划类的笔记“写好了”再按它们办事，有一件容易做的小事很快就可以动起来了
